# 참고 글 구조에서 가져온 패턴

## Yohan Studio 네이버 글

참고: https://blog.naver.com/yohan3071/224295484377

`크롬 확장 프로그램, 아이디어 4개를 합쳐 18일 만에 스토어 제출한 후기`에서 유지할 장점:

- 첫 문장에서 기간과 결과를 제시
- 비개발자와 AI 협업이라는 작성자 맥락 공개
- 문제, 제품 차이, 숫자, 타임라인, 기술 결정, 실패, 제출 과정을 순서대로 설명
- 짧은 문단과 이미지 구분으로 모바일 읽기 흐름 확보
- 성공만 쓰지 않고 실패와 해결을 함께 기록

개선할 점:

- 섹션마다 달라진 테스트 개수처럼 수치 드리프트 제거
- 원문·GitHub·사이트 링크 반복 축소
- 이미지 앞에 무엇을 확인할지 설명
- 웹 원문을 그대로 붙이지 않고 네이버 문단 호흡으로 재편집

## 제품 블로그

### Vercel v0

참고: https://vercel.com/blog/announcing-v0-generative-ui

`제품 정의 → 실제 관심 수치 → 작동 방식 → 제공 조건 → 다음 단계`

가져올 점: 도입에서 제품이 무엇인지 한 문장으로 정의하고, 긴 배경보다 작동 방식을 빠르게 보여준다.

### Warp Agent Mode

참고: https://www.warp.dev/blog/agent-mode

`출시 선언 → 할 수 있는 일 → 왜 중요한지 → 실제 예시 → 통제·보안 → 시작 방법`

가져올 점: 기능 목록만 나열하지 않고 실제 오류나 작업 예시를 이미지 가까이에 둔다.

### Google 사람 중심 콘텐츠 지침

참고: https://developers.google.com/search/docs/fundamentals/creating-helpful-content

가져올 점:

- 직접 경험과 독창적인 정보
- 내용을 정확히 설명하는 비과장 제목
- 작성 주체, 작성 방법, 작성 이유가 드러나는 글
- 검색 노출만을 위한 분량 늘리기 금지

## Yohan Studio 고정 차별점

경쟁사 문구를 복사하지 않는다. Yohan Studio 글은 다음 순서를 중심으로 쓴다.

`현실 문제 → 직접 실행한 증거 → 만든 과정 → 실패와 교훈 → 독자가 따라 할 다음 행동`
