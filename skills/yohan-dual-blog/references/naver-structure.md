> 실물 3편(`vhk-npm-cli-launch` · `dev-lectures-two-days` · `snapcontext`) + 기존 `channel-rules.md`·`content-model.md`에서 추출·확장.
> 이 파일 = 네이버 원고의 **고정/유연/진솔 슬롯 + 문체 + 이모지 + 주제 변형**의 SoT.
> 채널 공통(문단 길이·이미지·해시태그·발행 체크리스트)은 `channel-rules.md`, 12항목 뼈대는 `content-model.md`. 여기선 네이버 특화 구조만.

## 1. 슬롯 3종

### 고정 슬롯 — 모든 글 공통, 항상 유지 ("큰 틀")
| 슬롯 | 규칙 | 실물 예 |
|---|---|---|
| 제목 | 결과선행(대상+구체결과/기간). 과장·이모지·리스티클 금지 | "npm 패키지 배포: 바리스타가 첫 CLI 도구를 출시하기까지" |
| 첫 문장 | 결과·기간·맥락 1문장. **첫 5줄 안에 결과**(네이버 SEO) | "카페에서 일하는 비전공자가 AI와 함께 만든 CLI 도구를 npm에 공개했다." |
| 성공담 부정 | 도입에 실패 예고 1문장 | "이 글은 코딩을 잘해서 빠르게 만든 성공담이 아니다." |
| 바이브코딩 고정멘트 | 도입부 1줄(변주 허용). **스타일 = 가운데 정렬 + 회색 #c2c2c2, 이탤릭 없음** (2026-07-20 발행본 요한 수정 실측 — 변환기가 자동 적용) | `- 비전공자가 AI와 함께 만들고 직접 검증한 기록입니다. -` |
| 두괄식 결론 | 핵심 결론을 앞에 | "결론부터 말하면 …" |
| 이미지 마커 | `[이미지 삽입: 무엇 + 어디를 봐야 하는지]` 증거형 | "[이미지 삽입: … Current Tags의 2.5.1과 latest 표시를 확인]" |
| 하단 서명(맺음) | 고정 2문장, **존댓말** | "개발자가 아닌 사람이 AI와 함께 만드는 과정을 기록하고 있습니다. 부족한 부분도 많지만, 하나씩 만들어가며 남기려고 합니다." |
| 원문 백링크 | `yohanstudio.co/blog/<slug>`, 끝에 1번, UTM 부착(D2) | |
| 해시태그 | 5~8개(핵심키워드+성격+기술+대상) | "#특강후기 #개발자특강 #바이브코딩 …" |

### 유연 슬롯 — 주제별로 달라짐 (→ §5 주제 변형)
한눈에 요약 유무 · 소제목 스타일 · 도입 훅 · 중심축 · 여백(제로폭 공백) · 계절 인사.

### 진솔 슬롯 — ★사람이 직접 타이핑 (자동생성 금지)
- 개인 감정: "심장이 내려앉았다", "솔직히 일단 잤다. 머리가 복잡해서"
- 판단·의도·해석: 왜 이게 나한테 의미였는지
- 계절/상황 인사의 최종 문구(틀만 제공)
→ 변환기·AI는 이 슬롯을 `[여기 네 말: …]` 플레이스홀더로 **비워둔다**. 시간절약의 핵심 = 기계 슬롯만 자동, 진솔 슬롯은 사람.

## 2. 문체(레지스터) 규칙 — ★
- 본문 = 평서문 `-다`로 통일.
- 맺음 서명 = 존댓말(고정).
- **계절/도입 인사도 본문과 같은 `-다`로 쓴다.** (존댓말 인사 → -다 본문 인접 시 어색. 2026-07-13 사용자 피드백)
  - ✗ "…망설였습니다 ☀️ / …특강 3개를 들었다." (~니다/~다 혼재)
  - ✓ "요즘 낮 더위가 만만치 않다. …망설였다 ☀️ / …특강 3개를 들었다."

## 3. 계절/상황 인사 슬롯 (유연·선택)
- 위치: 맨 앞 **딱 1줄**(결과가 5줄 밖으로 밀리면 안 됨).
- 문체: 본문과 동일 `-다`. 최종 문구는 진솔 슬롯(사람).
- 틀 예시: 여름 "요즘 낮 더위가 만만치 않다." / 겨울 "며칠째 추위가 매섭다."
- 스티커(OGQ 등)는 사용자가 에디터에서 직접 삽입 — **스코프 밖**.

## 4. 이모지 규칙 — 브랜드 커스텀 아이콘 (2026-07-21 전환 · 이전 route A는 §4-이력)
- **소스 = 요한 브랜드 아이콘 8개** (오렌지 실루엣 PNG). SoT: `src/data/emojiSet.ts`, 생성: `scripts/gen-emoji.mjs`.
  - 전환 이유: 유니코드 이모지가 브랜드 톤과 안 맞는다는 요한 판단(2026-07-21). 웹(`EmojiIcon`)과 같은 아이콘을 써서 채널 간 얼굴을 통일한다.
  - 라이선스: Material Symbols · MDI 둘 다 Apache-2.0 (상업·재배포 허용, 표기 불요).

### 4-1. 네이버 삽입 규격 — SE ONE 실측 확정 (2026-07-21)
```
<img src="https://yohanstudio.co/images/emoji/<개념>-solid.png" width="16" height="16" />&nbsp;텍스트
```
| 항목 | 확정값 | 실측 근거 |
|---|---|---|
| 크기 | 본문 **16px** / 소제목 **22px** | 본문은 14px면 작고 17px부터 위로 뜬다. 소제목은 `<b>`로 굵어져 글자 무게가 커지므로 폰트 비례(20px)로는 눌린다 — 굵기 보정분 포함 22px (24px는 줄을 밀어냄) |
| 간격 | **`&nbsp;` 필수** | 일반 공백·`margin` 모두 붙여넣기에서 소실 |
| 위치 | **줄 맨 앞 전용** | 문장 중간 금지 — 아래 정렬 한계 때문 |
| 소스 | **외부 http URL만** | data URI img는 붙여넣기에서 제거됨 |
| 스타일 | **인라인 style 금지** | SE가 `style` 속성을 통째로 제거한다 |

- **정렬 한계(우회 불가)**: SE는 인라인 이미지를 정사각으로 강제하고 baseline에 고정한다. 그 결과 커스텀 아이콘은 유니코드 이모지보다 **3.6px 위**에 앉는다. `<sub>` 태그·`height≠width`·`valign` 속성·에디터 아래첨자 버튼 — 4가지 모두 실측했고 전부 무효였다. 그래서 **비교 대상이 생기는 문장 중간 사용을 금지**하고 줄 맨 앞에서만 쓴다(요한 확정).
- **자동 치환**: `naver-to-html.mjs`가 fragment 생성 시 줄 맨 앞 유니코드 이모지를 위 규격으로 바꾼다. 원고(.md)는 계속 유니코드로 쓰면 된다.
- **팔레트 18개 전부 커스텀으로 덮여 있다**(2026-07-21 확장). 팔레트 밖 이모지만 금지 — 변환기가 못 잡아 유니코드로 남고, 커스텀과 섞이면 톤이 어긋난다. 새 개념이 필요하면 `src/data/emojiSet.json`에 추가하고 이 표를 갱신한다.

### 4-이력 (2026-07-13 route A — 폐기)
- 당시 결정: "소스 = 유니코드 이모지만" (무료·표기0·붙여넣기 무마찰). 이미지 이모지는 매 글 업로드 수고 + "이미지=증거" 원칙을 흐린다는 근거.
- 폐기 사유: 브랜드 아이콘을 프로덕션에 상시 배포(`/images/emoji/*.png`)하게 되면서 "매 글 업로드" 전제가 사라졌다. URL 참조만 하므로 마찰 0.
- **원칙**: 장식 아님. 팔레트 안에서 의미가 맞는 이모지를 **섹션 제목당 기본 1개**(Q&A처럼 형식이 이미 명확한 섹션만 생략). 섹션 밖 본문에선 경고/강조 문단 앞 0~1개. 팔레트 밖 이모지 금지.
  - 2026-07-20 갱신: 기존 "생략 기본"이 밋밋하다는 요한 피드백(0.3.0 네이버 발행) — 소제목 예고형을 기본으로 상향. 실발행 선례: "⚠️ 삽질 1:" (npm 후기).
- **위치**: 소제목 · 핵심 리스트 항목 · 경고/강조 문단 앞.
- **요한 브랜드 팔레트** (같은 개념엔 항상 이 이모지 — 흐름 통일의 핵심):

  원고(.md)에는 아래 유니코드로 쓴다. 변환기가 전부 브랜드 아이콘으로 바꾼다.
  **팔레트 18개가 모두 커스텀으로 덮여 있어** 유니코드가 섞일 일이 없다.

  | 개념 | 원고 표기 | 키 | 개념 | 원고 표기 | 키 |
  |---|---|---|---|---|---|
  | 결과·수치·지표 | 📊 | `result` | 질문·고민 | ❓ | `question` |
  | 아이디어·깨달음 | 💡 | `idea` | 성공·완료·통과 | ✅ | `success` |
  | 삽질·실패 | 💥 | `fail` | 도구·CLI | 🛠️ | `tool` |
  | 배포·출시 | 🚀 | `launch` | 보안 | 🛡️ | `security` |
  | 위험·주의 | ⚠️ | `warning` | 배움·특강 | 🎓 | `learn` |
  | 버그 | 🐛 | `bug` | AI | 🤖 | `ai` |
  | 비전공·시작·성장 | 🌱 | `growth` | 시간·타임라인 | ⏱️ | `time` |
  | 여름 | ☀️ | `summer` | 겨울 | ❄️ | `winter` |
  | 봄 | 🌸 | `spring` | 가을 | 🍂 | `autumn` |

  팔레트 **밖** 이모지는 여전히 금지 — 변환기가 못 잡아 유니코드로 남고, 커스텀과 섞이면 톤이 어긋난다.

- 예: 소제목에 '보안' 글자가 없으면 🛡️로 예고. 이미 '보안'이 글자로 있으면 **생략이 기본**(원하면 맨 앞 예고용으로 유지 가능 = 취향).

## 5. 주제 변형 (topic variants)
| 슬롯 | 제품/빌드로그 (출시·도구) | 회고/후기 (특강·행사·경험) |
|---|---|---|
| 한눈에 요약 | 있음(도입 뒤) | 없음(바로 서사) |
| 소제목 | 질문형("~는 뭔가?", "왜 직접 만들었나?") | 서사·시간순("금요일 낮 — …") |
| 도입 훅 | 담백한 결과선언 | 감정 서사(후회·두려움·궁금증) |
| 중심축 | 정의→증거→과정→사용법→현재상태 | 상황→결정→교훈→내 변화 |
| 이미지 | 실행·결과 화면(증거) | 현장 증거 + 웹 섹션 캡처 |
| 이모지 톤 | 📊✅🚀🔧 | 🛡️💡✋❓ |

→ **개발/IT 특강후기 = 회고형 베이스 + "왜 갔나 → 무엇을 배웠나 → 어떻게 적용" 3축** (B2 첫 템플릿과 연결).
