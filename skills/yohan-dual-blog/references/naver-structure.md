# 네이버 글 구조 스펙 (Yohan Studio)

> 실물 3편(`vhk-npm-cli-launch` · `dev-lectures-two-days` · `snapcontext`) + 기존 `channel-rules.md`·`content-model.md`에서 추출·확장.
> 이 파일 = 네이버 원고의 **고정/유연/진솔 슬롯 + 문체 + 이모지 + 주제 변형**의 SoT.
> 채널 공통(문단 길이·이미지·해시태그·발행 체크리스트)은 `channel-rules.md`, 12항목 뼈대는 `content-model.md`. 여기선 네이버 특화 구조만.

## 1. 슬롯 3종

### 고정 슬롯 — 모든 글 공통, 항상 유지 ("큰 틀")
| 슬롯 | 규칙 | 실물 예 |
|---|---|---|
| 제목 | 결과선행(대상+구체결과/기간). 과장·이모지·리스티클 금지 | "npm 패키지 배포: 바리스타가 첫 CLI 도구를 출시하기까지" |
| 첫 문장 | 결과·기간·맥락 1문장. **첫 5줄 안에 결과**(네이버 SEO) | "카페에서 일하는 비전공자가 AI와 함께 만든 CLI 도구를 npm에 공개했다." |
| 성공담 부정 | 도입에 실패 예고 1문장 | "이 글은 코딩을 잘해서 빠르게 만든 성공담이 아니다." |
| 바이브코딩 고정멘트 | 도입부 1줄(변주 허용) | `- 비개발자가 AI와 함께 만들고 직접 검증한 기록입니다. -` |
| 두괄식 결론 | 핵심 결론을 앞에 | "결론부터 말하면 …" |
| 이미지 마커 | `[이미지 삽입: 무엇 + 어디를 봐야 하는지]` 증거형 | "[이미지 삽입: … Current Tags의 2.5.1과 latest 표시를 확인]" |
| 하단 서명(맺음) | 고정 2문장, **존댓말** | "개발자가 아닌 사람이 AI와 함께 코딩에 도전하고 있습니다. 부족한 부분도 많겠지만, 하나씩 만들어가는 과정을 기록하려고 합니다." |
| 원문 백링크 | `yohanstudio.co/blog/<slug>`, 끝에 1번, UTM 부착(D2) | |
| 해시태그 | 5~8개(핵심키워드+성격+기술+대상) | "#특강후기 #개발자특강 #바이브코딩 …" |

### 유연 슬롯 — 주제별로 달라짐 (→ §5 주제 변형)
한눈에 요약 유무 · 소제목 스타일 · 도입 훅 · 중심축 · 여백(제로폭 공백) · 계절 인사.

### 진솔 슬롯 — ★사람이 직접 타이핑 (자동생성 금지)
- 개인 감정: "심장이 내려앉았다", "솔직히 일단 잤다. 머리가 복잡해서"
- 판단·의도·해석: 왜 이게 나한테 의미였는지
- 계절/상황 인사의 최종 문구(틀만 제공)
→ 변환기·AI는 이 슬롯을 `[여기 네 말: …]` 플레이스홀더로 **비워둔다**. 시간절약의 핵심 = 기계 슬롯만 자동, 진솔 슬롯은 사람.

## 2. 문체(레지스터) 규칙 — ★
- 본문 = 평서문 `-다`로 통일.
- 맺음 서명 = 존댓말(고정).
- **계절/도입 인사도 본문과 같은 `-다`로 쓴다.** (존댓말 인사 → -다 본문 인접 시 어색. 2026-07-13 사용자 피드백)
  - ✗ "…망설였습니다 ☀️ / …특강 3개를 들었다." (~니다/~다 혼재)
  - ✓ "요즘 낮 더위가 만만치 않다. …망설였다 ☀️ / …특강 3개를 들었다."

## 3. 계절/상황 인사 슬롯 (유연·선택)
- 위치: 맨 앞 **딱 1줄**(결과가 5줄 밖으로 밀리면 안 됨).
- 문체: 본문과 동일 `-다`. 최종 문구는 진솔 슬롯(사람).
- 틀 예시: 여름 "요즘 낮 더위가 만만치 않다." / 겨울 "며칠째 추위가 매섭다."
- 스티커(OGQ 등)는 사용자가 에디터에서 직접 삽입 — **스코프 밖**.

## 4. 이모지 규칙 — 의미 기반·절제
- 원칙: 장식 아님. **의미가 맞을 때만**, 섹션당 0~1개, 남발 금지(AI 티 방지).
- 위치: 소제목 · 핵심 리스트 항목 · 경고/강조 문단 앞.
- 소스: **유니코드 이모지만**(무료·붙여넣기 무마찰). 오픈소스 이미지셋·OGQ 스티커는 스코프 밖(스티커=사용자 수동).
- 의미 매핑(가이드, 강제 아님):

  | 의미 | 이모지 | 의미 | 이모지 |
  |---|---|---|---|
  | 위험·주의 | ⚠️ | 결과·수치·지표 | 📊 |
  | 보안 | 🛡️ | 아이디어·깨달음 | 💡 |
  | 성공·완료·통과 | ✅ | 배포·출시 | 🚀 |
  | 실패·삽질·버그 | 🐛 / 🔧 | 시간·타임라인 | ⏱️ |
  | 질문·고민 | ❓ | 계절(여름/겨울) | ☀️ / ❄️ |

- 예: "실패와 해결" → "⚠️ 삽질 4: 보안 토큰이 그대로 노출된 사고"

## 5. 주제 변형 (topic variants)
| 슬롯 | 제품/빌드로그 (출시·도구) | 회고/후기 (특강·행사·경험) |
|---|---|---|
| 한눈에 요약 | 있음(도입 뒤) | 없음(바로 서사) |
| 소제목 | 질문형("~는 뭔가?", "왜 직접 만들었나?") | 서사·시간순("금요일 낮 — …") |
| 도입 훅 | 담백한 결과선언 | 감정 서사(후회·두려움·궁금증) |
| 중심축 | 정의→증거→과정→사용법→현재상태 | 상황→결정→교훈→내 변화 |
| 이미지 | 실행·결과 화면(증거) | 현장 증거 + 웹 섹션 캡처 |
| 이모지 톤 | 📊✅🚀🔧 | 🛡️💡✋❓ |

→ **개발/IT 특강후기 = 회고형 베이스 + "왜 갔나 → 무엇을 배웠나 → 어떻게 적용" 3축** (B2 첫 템플릿과 연결).
