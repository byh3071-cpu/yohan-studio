# Flexible Seat Randomizer

실제 현장에서 종이 제비 누락과 재추첨으로 2~3분 걸리던 13명 자리배치를 10초 이내 흐름으로 바꾼 프로젝트의 **포트폴리오용 익명 재구축판**입니다.

## 원본과의 관계

- 원본 MVP 제작 기록: 2026-01-28, 약 30분
- 이 저장소: 개인정보를 제거하고 기술 선택·테스트를 설명하기 위해 다시 만든 독립 데모
- 재구축판의 React·Vite 코드와 테스트 결과를 원본 MVP의 기술스택으로 주장하지 않습니다.

## 왜 서버와 DB가 없나요?

이 도구의 핵심은 당일 출석과 좌석을 한 번 배치하는 것입니다. 장기 저장이 핵심 요구사항이 아니며 실제 이름을 서버에 남기면 개인정보 위험만 커집니다. 따라서 모든 입력은 React 메모리에만 존재하고 새로고침 시 삭제됩니다.

검토한 대안:

- `localStorage`: 재접속은 편하지만 공용 기기에 이름이 남음
- Supabase: 히스토리를 만들 수 있지만 인증·권한·삭제 정책이 필요함
- 현재 선택: 무저장 정적 앱. 기능은 좁지만 공개 데모의 안전성과 설명 가능성이 높음

## 주요 동작

- 가상 고정 인원 2명 + 랜덤 배치 인원 13명
- 출석 상태와 배정 포함 여부를 별도 관리
- 고정 좌석 유지, 결석자 제외, 중복 배정 방지
- Fisher–Yates 셔플 + Web Crypto 기반 난수
- 서버 전송 없는 결과 PNG 생성

## 개인정보 원칙

- 기본 데이터는 모두 가상 인물
- 서버·DB·분석 SDK 없음
- 브라우저 저장소 미사용
- 새로고침 시 입력 삭제
- 이미지 생성은 브라우저 안에서만 수행

## 한계

- 원본 프로젝트 소스가 아니라 익명 재구축판입니다.
- `2~3분 → 10초 이내`는 당시 현장 측정 기록이며 장기 분석 통계가 아닙니다.
- 랜덤 결과에 대한 장기 만족도와 재작업률은 수집하지 못했습니다.
