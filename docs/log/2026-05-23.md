## 교훈

- vibrant 브랜드 색을 light bg 위 텍스트로 쓰면 WCAG AA는 물리적으로 통과 불가. dual token 패턴(`--brand` vs `--brand-text`)이 가장 깔끔한 우회 (→ `docs/patterns/css-vibrant-brand-vs-wcag-aa.md` 등록)
- 자동 검수 agent에서 위반 카운트만 저장하면 추적 불가능. **노드 디테일(target/fg/bg/ratio)을 처음부터 캡처**해야 액션 가능
- agent 발견 → 사람 판단 → 토큰 수정 → 사람 재검증 루프가 현실적. 완전 자동 수정은 다음 단계 (Phase 3+)
