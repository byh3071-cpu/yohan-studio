### 핵심 트러블슈팅 — `/store` "상품 목록을 불러오지 못했습니다"
- **증상**: DB엔 `studio_products` 3건 정상인데, 배포 사이트 `/store`는 SELECT 실패 에러박스
- **진단(전부 정상 배제)**: 코드(`supabase.ts` Proxy가 env 없으면 throw → catch가 에러박스) ·
  RLS(`studio_products_public_read`, `active=true`, anon 허용) · URL(ref `olacbbf…` 양쪽 일치)
- **원인**: anon key 값 / `NEXT_PUBLIC_*` 빌드 인라인 미반영
- **해결**: anon key 재등록 + **빌드 캐시 끄고 Redeploy** → `/store` 상품 3건 정상 복구
- **교훈**: `NEXT_PUBLIC_*` 는 빌드타임 인라인 → 값이 맞아도 캐시 빌드면 옛 `undefined` 잔존.
  값 변경 후엔 "Use existing Build Cache" 해제하고 재배포해야 반영됨.
