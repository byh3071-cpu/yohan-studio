## 트러블슈팅 / 교훈

1. **인라인 style display 가 CSS @media 를 specificity로 이긴다** — 미디어 쿼리로 display 토글하는 컴포넌트는 인라인 style에 display를 두면 안 됨. 클래스로만 처리.
2. **Edge headless `--screenshot` 은 viewport 캡처** — 풀페이지 캡처는 `--window-size`를 페이지 height만큼 키워야 하는데 ~3000px 이상에선 실패. 풀페이지 캡처 필요 시 puppeteer/playwright 같은 CDP 도구가 정답.
3. **Source Serif 4 variable 이름 충돌** — `variable: "--font-serif"` 로 next/font 등록하면 globals.css `@theme`에서 `--font-serif: var(--font-serif), ...` 형태 cycle. next/font variable은 `--font-source-serif` 같은 raw 이름을 쓰고, alias만 `--font-serif`로 노출하는 게 안전.
