| 후보 | 판정 | 근거 요약 |
|---|---|---|

| ShotGrade | 차순위 후보 (보류) | 기술 깊이는 최강(RLS·OAuth·AI 비전)이나 수치·실사용 근거 약함, 테스트 0건. 문제 서사가 본인 취미에 가까움. 매출·재고 다음 순번의 "기술 깊이 카드"로 유지 |

## 선정 근거 (현직자 조언 대비)

3인 현직자 공통 기준 — 왜 그렇게 설계했는지 설명 + 검증 + 운영 결과 책임 — 에서
매출·재고 자동화만 유일하게 "실제 운영 1개월 + 측정된 개선 + 이해관계자 피드백"을 모두 갖췄다.
기술 스택의 화려함(ShotGrade)보다 문제 해결 서사의 완결성을 우선한다는 쇼룸 원칙과 일치한다.

## 선정 프로젝트 완성 게이트 (게시 전 통과 조건)

1. **코드 회수**: script.google.com에서 GAS 원본 확보 → 전용 GitHub 저장소로 이전 (요한 수동 작업)
2. 실데이터 제거: 더미 매출 데이터 구글시트 + 익명 스크린샷 재구성 (플렉시블 익명 재구축 방식 재사용)
3. 직원 호칭·기관 상세명 익명화 (공개 금지 조항 준수)
4. 수치 3종에 측정 기준 명시: "당시 현장 측정 기록" 표기 원칙 유지
5. clasp 기반 로컬 개발 환경 재구성 + 핵심 집계 로직 단위 테스트
6. 쇼룸 flagship 케이스 스터디 데이터 계약(metrics·decisions·evidence) 충족

## 리스크

- GAS 코드가 회수 불가하면(계정 접근·유실) 선정을 ShotGrade로 교체한다.
- Notion 기록의 스크린샷은 "추가 예정" 상태 — 증거 이미지를 새로 만들어야 한다.
