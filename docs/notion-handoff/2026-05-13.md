## 3. 에러 × 1

### 🚨 TS-001 — React 19 hydration mismatch + script tag warning

| 필드 | 값 |
|---|---|
| 제목 | `TS-001 — React 19 hydration mismatch + script tag warning (다크모드 첫 구현)` |
| 유형 | 에러 |
| 상태 | Resolved |
| 날짜 | 2026-05-13 |
| 출처 세션 | 세션 2 |
| 파일 | `docs/troubleshooting/001-hydration-mismatch-and-script-warning.md` |
| 심각도 | High (콘솔 에러 + UX 깜빡임) |

**증상**: `<script dangerouslySetInnerHTML>` React 19 워닝 + ThemeToggle 글리프(☾↔☀) 서버/클라이언트 마크업 불일치

**원인**: ① 인라인 script가 React 트리 안 ② `useState(readInitial)` 가 SSR=light/CSR=dark 분기 ③ 첫 렌더 분기 글리프 즉시 노출

**해결**: ① `next/script beforeInteractive` 로 head hoist ② useState 초기값 항상 `'light'` 고정 (mount 후 sync) ③ `<html suppressHydrationWarning>` + ThemeToggle hydrated placeholder

**교훈** (요약): React 19에서 inline script는 `next/script beforeInteractive` 필수 / 클라 전용 상태는 SSR 보수적 단일값 고정 / DOM attribute는 `useSyncExternalStore + MutationObserver` 가 최적 / `suppressHydrationWarning` 은 root 전용

## 4. TIL × 7

> 노션엔 한 줄씩 7개 row 로 적재. 모두 출처: 세션 3 (2026-05-13).

| # | 한 줄 |
|---|---|
| 1 | `useSyncExternalStore` 는 React state 만이 아니라 DOM attribute / localStorage / window 등 외부 소스 단일 구독 API. `subscribe` + `getSnapshot` + `getServerSnapshot` 로 SSR/CSR 양쪽 hydration 안전. |
| 2 | `useEffect` 안의 setState는 cascading render 일으켜 `react-hooks/set-state-in-effect` 룰 차단. 콜백(observer/timer/event)의 setState는 외부 이벤트 응답이라 허용. effect **본문 동기 호출**만 룰 대상. |
| 3 | prop 변경 시 자식 state 초기화는 `useEffect` 아니라 **render 중 setState + prev-prop 비교**가 React 공식. `if (current !== prev) { setPrev(current); setChildState(initial) }` — 같은 컴포넌트 호출은 React가 commit 전 batch. |
| 4 | `next/image` `priority` 속성은 LCP 후보에 명시 → 자동 `loading="eager"` + `fetchpriority="high"`. 카드 리스트 첫 카드(`index === 1`)에만 거는 게 표준. |
| 5 | 외부 배포 데모/포트폴리오에 **실명 등 개인정보** 노출 시, 익명화 재배포 전까지 demo URL 즉시 제거가 안전. 코드 주석에 사유·복원 조건 남겨야 다음 세션 재등록 실수 방지. |
| 6 | JSX 자식 `// 텍스트` 가 그대로 노출되면 `react/jsx-no-comment-textnodes` 차단 — `{`\`// ${value}\``}` 또는 `{"// 텍스트"}` 로 텍스트노드 명시. |
| 7 | ES2019 부터 `try { ... } catch { ... }` (binding 생략) 가능 — unused 변수 경고 회피용으로 `catch (e)` → `catch` 단순화. |
