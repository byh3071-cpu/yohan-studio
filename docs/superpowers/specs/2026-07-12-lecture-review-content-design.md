# 특강 후기 콘텐츠 설계 — 노션 3편 + 블로그 통합 1편

- 작성일: 2026-07-12
- 상태: **설계 승인 대기 → 사용자 이탈로 세션 중단. 설계 내용은 사실상 합의 완료, 미결 1건(노션 3편 위치)만 남음**
- 재진입 지점: 이 문서 전체 읽기 → "미결 사항" 확인 → "다음 액션" 순서대로 실행

## 1. 배경과 의도 (사용자 원래 요청)

백요한이 2026-07-10(금)~11(토) 서울청년정책박람회(DDP)와 청년쓰리룸에서 현직 개발자 특강 3건을 들었다. 목적:

- 특강 후기를 **포트폴리오에 도움되는 형태**로 콘텐츠화 — 왜 갔는지 / 무엇을 들었는지 / 어떻게 생각했는지 명확하게
- 노션(AI 노트·리소스·서머리)을 요한 스튜디오 웹사이트에 **연동**해서 자세한 내용은 노션에서 보게
- 현업 개발자한테도 어필 + 일반인이 유입돼도 쉽게 이해되는 글
- **AI가 써준 티 안 나게** — 본인 생각·의도 100% 반영 (질문 인터뷰로 채움)

## 2. 확정된 결정 사항

| # | 결정 | 내용 |
|---|---|---|
| 1 | 연동 전략 = C→B | 지금은 A(블로그 글 + 노션 링크)로 발행, 다음 스프린트에 B(Notion API 연동 "러닝 로그" 라우트) 구축. **최종 목적지는 B 확정**, 순서만 A 먼저 |
| 2 | B 점진 로드맵 | B v1 텍스트·리스트·코드 렌더 → B v2 이미지(ISR 또는 프록시) → B v3 Supabase Storage 리호스팅. 이미지 URL 만료는 Notion 영구 스펙이지만 우리 쪽에서 해결 가능 |
| 3 | 콘텐츠 구조 = 3+1 | 노션에 특강별 후기 3편(각자 완결, 공통점 추출 아님) + 블로그에 통합 1편(시간순 서사) |
| 4 | 통합 1편 주제 | "AI가 다 해주는 시대에 결국 남는 건 왜 이걸 했는지 설명하고 책임지는 사람" — 단, 세 명이 같은 말을 한 게 아니라 **AI라는 같은 파도를 세 각도에서 본 것**으로 구성 (공통점 나열 금지) |
| 5 | 표기 규칙 | 실명 전부 제외: "IBM 보안 스페셜리스트" / "AI 기업 현직 개발자"(쓰리룸) / "다니엘(닉네임)"(메타코드). 노션 3편도 공개될 것이므로 동일 적용 |
| 6 | 제작 주체 | 노션 3편은 Claude가 notion-mcp로 직접 생성 (인터뷰 내용 보유). 블로그 1편은 studio-post 스킬 절차로 저작 |

### C→B 결정 근거 (요약)

- 특강 후기는 신선도가 생명 → 글은 기존 MDX 파이프라인으로 즉시 발행
- 현재 배포 블로커(Vercel env 미세팅·Supabase 마이그레이션 미실행) 위에 NOTION_API_KEY 블로커를 얹지 않음
- B는 "이 글의 부속 기능"이 아니라 **"러닝 로그 시스템"이라는 독립 기능**으로 격상 — 특강 후기가 첫 콘텐츠가 되고 이후 모든 배움이 같은 파이프라인을 탐
- 글 안에 "이 노트는 내 Notion 세컨드브레인에서 관리된다 → 곧 사이트에서 바로 보게 만들 예정" 예고 심기 → A→B 전환 자체가 스토리

### 노션 3편 공통 템플릿 (각자 완결)

1. 한 줄 결론 (두괄식)
2. 왜 갔나 — 디스코드 크롤링 알림으로 발견한 경위 + 기대와 두려움
3. 현장 — 장소·분위기·아쉬운 점 (박람회 소음 비판 포함)
4. 무엇을 들었나 — 토픽별 정리 (기존 SUMMARY에서 발췌, 파인만식으로 쉽게 풀어씀)
5. 기억에 남는 말 — 본인 기억 속 표현 그대로
6. 내가 던진 질문과 받은 답 (쓰리룸 편은 "못 물어본 질문" → 스스로 답한 과정)

9. 원본 링크 — 기존 상세 SUMMARY 페이지 (딥다이브용)

## 4. 인터뷰 원자료 (백요한 본인 목소리 — 글에 반드시 반영)

> AI 냄새 제거의 핵심 재료. 아래 표현·감정·디테일을 각 글에 살릴 것.

### 왜 갔나 / 신청 경위

- 온통청년·정보퐁퐁 공고를 평소 자주 확인 + **각 사이트 API를 활용한 크롤링으로 디스코드 알림**을 받아서 알게 됨 (박람회·쓰리룸 둘 다). ← 개발자 어필 디테일, 글 도입부 후보
- 개발자를 꿈꾸면서 방향·공부·학습·취준 준비를 어떻게 할까 관심 갖던 중 신청
- 기대: 비전공자 바이브코더를 현직자는 어떻게 생각할지 궁금했음
- **당일에 가려니 두려웠음** — AI로 바이브코딩하는 것을 안 좋게 보거나 비전공자라서 무시당할까 봐. ← 감정 아크의 시작점

### 두려움의 결말 (감정 아크 종착)

- 첫 특강(보안 스페셜리스트) 이후 조금씩 해소 → 2번째, 3번째로 갈수록 점점 해소
- 마지막 메타코드 특강 후: 내가 내뱉은 문제들의 **키워드를 나열해서 하나씩 해보는 것**이 중요하고, 혼자 프로젝트·개발할 때 명확하게 알아야 하며 그걸 **설명할 줄 아는 게 중요**하다고 정리됨
- 결론: "두려움보다는 **생각이 명확해진 계기**가 됐다"

### 세션별 기억·평가

**① 박람회 IT 멘토링 (7/10 금 11:40~12:20, DDP 아트홀 1관)**
- 예상과 달랐음: IT/개발인 줄 알고 갔는데 보안 스페셜리스트였음 → 오히려 흥미로워서 다른 데 안 가고 40분 유심히 들음
- 가장 인상 깊은 것: 현직자의 태도와 마인드. **물류·배민에서 일하다가 보안으로 전환한 이야기** — "남을 도와주는 사람이 되고 싶다"는 목표는 같고, 물류든 배민이든 잘 포장해서 보내주면 받는 사람이 도움받는 것처럼 보안도 누군가를 도와주는 일. **"일의 형태만 다르지 되고자 하는 사람은 같다"** ← 통합 1편 감성 축
- 직접 질문함: "인프런 무료 네트워크 강의 듣다가 보안 중요성을 다시 알게 됐는데, 보안 기초지식·공부를 어떻게 넓힐까요?" → 답: "보안 기초는 인프런·유튜브 강의로도 충분할 것"
- 불만: **주최측 세팅 아쉬움** — 행사장 앞 테이블이라 소음 크고 말이 가끔 묻힘. 개선 바람. ← 솔직 비판 한 줄로 글에 반영

**② 청년쓰리룸 현직개발자 특강 (7/10 금)**
- 디스코드 크롤링 알림으로 알게 됨. 인원 15~20명, **비전공자가 절반 정도** (본인 포함, "꽤 많다"고 함). 공간 넓고 깨끗하고 깔끔해서 듣기 좋았음. 강사 느낌: 명확하게 알려줌, "낫배드"
- 기억: 특별한 한 방보다 전체적으로 명확한 강의. **신입/경력을 정확하게 나눠 설명**해준 것이 기억에 남음
- 핵심 장면: Q&A에서 어떤 교육자가 "웹 런칭 준비 중인데 Vercel+Supabase로 배포한다" 하니 강사가 **"그건 비춘데?"** → 그 자리에서 "왜 비추예요?"라고 못 물어본 게 아쉬웠음. 왜 비추인지, 왜 AWS를 써야 하는지 궁금했음
- **후속**: 이 질문을 집에 와서 스스로 파고들어 노션에 교차검증 문서를 만들었음 (플랫폼 자체보다 설정·검증 능력이 핵심이라는 결론) ← "현장에서 못 물어본 질문에 스스로 답했다" = 개발자 어필 베스트 장면

**③ 메타코드 AI 현직자 특강 (7/11 토, DDP 아트홀 1관, 1시간)**
- 셋 중 **제일 좋았음** — 내용이 명확하고 확실하게 알려주는 느낌, 조언도 좋았음
- **직접 질의응답한 내용**: "AI로 개발을 시작했다 보니 원초적 지식·기본기가 중요하다고 생각하는데 배우고 습득하기 쉽지 않다. 보안이 가장 중요하고 책임져야 한다고 생각하는데 AWS를 쓰자니 비용이 감당 안 된다" → 다니엘 답: "이미 문제를 충분히 얘기한 것 같다. **본인이 키워드를 잘 나누고 나열해서 하나씩 해보면 좋겠다.** A-Z까지 다루는 강의나 유튜브 영상을 한번 봐라. **좀만 하시면 금방 잘하실 것 같다**" ← 두려움 해소 장면

### 듣고 나서 실제로 한 것 (적용 증거)

1. **일단 잤음 ㅋㅋ** — 머리가 복잡해서. 현장에서는 노션 AI 노트 → 리소스 → 서머리 요약까지 했는데 본인 생각·의견 정리가 안 잡혀서 일단 잠 ← 솔직 디테일, 글에 그대로
2. 자고 일어나서 **요한 스튜디오 쇼룸 정비** — "왜 이걸 했는지, 왜 이 기술 스택을 썼는지 명확하게 설명할 줄 알아야 한다"는 공통 교훈을 듣고 프로젝트 설명을 다시 씀 ← 핵심 적용 사례
3. **AI 사용 습관 변화** — AI가 좋아질수록 알아서 다 해버리고 스킵하게 되니까, 프롬프팅을 자세하고 구체적으로 하고, 짚고 넘어가고 지적하면서 놓치지 않으려 함
4. (노션에 이미 존재) **VHK Secure 보안 자동화 설계** 문서 — 특강 교훈이 제품 설계로 이어진 증거
