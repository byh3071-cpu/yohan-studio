### 이번에 새로 획득한 기술 자산 (함정 포함)

- **PS 5.1 `Set-Clipboard -AsHtml` = ANSI 인코딩 버그** → 한글 전멸. CF_HTML을 UTF-8 바이트 오프셋으로 직접 빌드해야 함. 해결 스크립트: `scripts/Set-ClipboardHtmlUtf8.ps1` (레포 영구화). 패턴 등재: `docs/patterns/PAT-004-cf-html-clipboard-utf8.md`
- SmartEditor는 **합성 paste 이벤트 무시** (isTrusted 체크) → 네이티브 Ctrl+V 필요. 자동화는 playwright 키 입력으로 가능
- 네이버 에디터 업로드는 playwright filechooser 인터셉트로 자동화 가능 (커서 위치에 정확 삽입)
- 발행 목록·날짜는 `m.blog.naver.com/api/blogs/<id>/post-list`로 기계 확인 가능 (수기 기억 불필요)

## 4. 반자동 도구 요구사항 (확정)

이름(가칭): **naver-convert** — "네이버 변환해줘" 트리거

1. 입력: MDX slug (yohanstudio.co 발행글)
2. LLM 의미 변환: 채널 규칙 적용 (문단 1~3문장 분절 · 용어 한국어 풀이 · 제로폭 공백 여백 · 이미지 위치 마커 · 해시태그 5~8개)
3. 네이버 친화 HTML 생성 → CF_HTML UTF-8 클립보드 적재 (`Set-ClipboardHtmlUtf8.ps1` 내장)
4. 라이브 블로그 섹션 자동 캡처 → `exports/<slug>/img/NN.png` (마커 번호 매칭)
5. 발행 후: 사람이 URL 입력 → 콘텐츠 허브 자동 기록 (채널=네이버, 상태=발행)
6. 사람 게이트 유지: 붙여넣기·최종 편집·발행 버튼 = 사람 (콘텐츠 OS 🟡 정의 준수. 🔴 발행 API 자동화는 하지 않음)

- 위치: **yohan-cc-skills 플러그인** (레포 skills/ 아님 — 양 PC 동기화 + 프로젝트 무관 사용)
- 참조 자산: 미머지 브랜치의 `channel-rules.md`·`mdx-to-naver.mjs`는 체리픽 참조 후 플러그인으로 흡수 (브랜치 자체는 머지 안 함)
