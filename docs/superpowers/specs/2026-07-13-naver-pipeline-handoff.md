## 3. 갭 분석 — "개고생"의 원인 4개

1. **자산이 미머지 브랜치에 갇힘**: 스킬·스크립트·규칙 전부 `fix/vhk-blog-v251-captures`에만 있어서 master 기반 세션에서 안 보임/못 씀. 이번 세션도 브랜치에서 git show로 꺼내 읽었음
2. **기계 변환의 반쪽짜리**: mdx-to-naver.mjs는 마크다운 문법 제거만 함. 문단 재구성(1~3문장 분절)·용어 한국어 풀이·모바일 줄나눔·네이버식 여백(​ 제로폭 공백) 같은 의미 작업은 전부 수동 — 스킬 스스로 "자동 변환 결과 그대로 발행 금지"라고 명시
3. **서식 손실**: 평문 .txt를 네이버 에디터에 붙여넣으면 굵게·인용·구분선이 다 사라져서 에디터에서 서식을 손으로 다시 입힘 ← 이게 최대 개고생 포인트
4. **이미지 수작업**: 블로그 섹션 캡처를 사람이 일일이 찍고 배치. 어젯밤 headless Chrome 캡처로 자동화 가능성 확인됨 (`chrome --headless=new --screenshot`)

## 5. 개선안 옵션 (브레인스토밍 출발점 — 결정은 아침에)

| 옵션 | 내용 | 효과 | 난이도 |
|---|---|---|---|
| A. 스킬 정식화 | 브랜치의 yohan-dual-blog를 **yohan-cc-skills 플러그인으로 이관**(양 PC 동기화) + LLM 의미 변환(문단 분절·용어 풀이·제로폭 여백)을 스킬 절차에 정식 포함. 트리거 "네이버 변환" | 변환 품질 일관 + 어디서든 사용 | ★★ |
| B. HTML 클립보드 | 네이버 SmartEditor는 HTML 붙여넣기 시 서식(굵게·인용·구분선) 유지됨 → MDX를 네이버 친화 HTML로 변환 + PowerShell `Set-Clipboard`(HTML 포맷)로 복사 → **붙여넣기 한 방에 서식 완성**. 갭 3 직격 해결 | 서식 수작업 제거 | ★★ (에디터 새니타이저 실험 1회 필요) |
| C. 이미지 자동 캡처 | 라이브 블로그를 headless Chrome으로 섹션별 자동 캡처 → `exports/<slug>/img/01.png...` 생성, 원고의 [이미지 N] 마커와 번호 매칭 | 캡처 수작업 제거 | ★ (어제 검증된 방법 재사용) |
| D. 발행 API 자동화 | 네이버 블로그 글쓰기 API(OAuth) 직접 발행 | 완전 자동 | ⚠️ 비추천 — 콘텐츠 OS 계단 위반(반자동 건너뜀) + 외부 발송 사람 게이트 원칙. 🔴 칸에서 재론 |

**추천 조합 [추론]**: A+B+C를 하나의 스킬로 — "네이버 변환해줘" → ① LLM 의미 변환(A) ② HTML 클립보드 복사(B) ③ 이미지 자동 캡처(C) ④ 사람은 붙여넣기+이미지 드래그+발행 버튼만. 콘텐츠 OS 🟡 반자동 정의("변환은 코드, 승인·발행은 사람")와 정확히 일치.

**선결 확인 (아침 브레인스토밍 전 사람 확인 1개)**: B의 전제 — 네이버 에디터에 HTML 클립보드 붙여넣기 시 어떤 서식이 살아남는지 실험 1회 (5분. 실험용 HTML은 세션이 만들어줌)
