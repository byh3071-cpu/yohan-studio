# 디스콰이엇 메이커로그

## 제목
바리스타의 CLI, npm 배포까지 — "다 됐어요"를 못 믿어서 만든 도구

## 본문

이번 진전: 내가 만든 CLI 도구 VHK가 npm에 올라갔다. `npm i -g @byh3071/vhk` 한 줄로 누구나 설치할 수 있는 상태.

과정을 좀 풀면 — 나는 카페에서 일하는 비전공자고, 밤에 AI랑 코딩한다. 만들면서 제일 크게 막힌 건 기술이 아니라 신뢰 문제였다. AI가 "다 됐어요"라고 하는데 실행하면 안 돌아가는 일이 반복됐고, 한 번은 그걸 찾느라 반나절을 통째로 날렸다.

그때 내린 결정이 이 프로덕트의 정체성이 됐다: 실수를 줄이려고 다짐하지 말고, 실수가 불가능한 구조를 만들자. 그래서 VHK는 테스트를 통과하지 못하면 "완료" 자체를 거부한다(vhk check). 목표도 한 번에 하나만 잡게 강제한다(vhk goal, 1 PR = 1 goal).

배포에서 배운 것 — npm 배포는 패키지 이름 규칙, 버전, 2단계 인증(OTP)이라는 또 다른 산이었다. 인증이 필요한 배포 단계는 일부러 자동화하지 않고 사람 몫으로 남겼고, 그 직전 점검만 vhk preflight로 묶었다. 사람이 해야 하는 것과 도구가 해야 하는 것의 경계를 정하는 게 자동화의 핵심이라는 걸 몸으로 배웠다.

다음 계획: 이 도구로 내 다음 프로덕트(콘텐츠 멀티플랫폼 변환기)를 개발하면서 독푸딩 데이터를 쌓는 것. 도구가 도구를 만드는 루프를 돌려본다.

연결 프로덕트: VHK (v2.5.1 · 테스트 1162 pass · 목표 문서 47개)
