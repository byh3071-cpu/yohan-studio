# X 스레드 (6트윗)

## 1/
바리스타가 만든 CLI가 npm에 올라갔다.

코딩 지식 0에서 시작해 테스트 1162개 통과, 목표 문서 47개까지.

비전공자가 AI랑 도구를 출시하며 배운 것 4가지:

## 2/
1. 한 번에 하나만

큰 걸 통째로 만들려 하면 무조건 멈춘다.
명령어 딱 하나 동작시키기부터 시작했다.

1 PR = 1 goal. 이 규칙을 아예 도구에 박았다.

## 3/
2. "이 줄 뭐야?"를 계속 물어라

코드를 못 읽어도 된다.
AI가 설명하고, 나는 판단만 한다.

한 번에 다 이해하려 하면 포기하게 된다. 이해는 질문으로 차오른다.

## 4/
3. "됐다"를 믿지 마라

AI의 "다 됐어요" 믿었다가 반나절 디버깅으로 갚았다.

해법은 다짐이 아니라 구조 — 테스트 실패면 도구가 완료 자체를 거부하게 만들었다.

실수는 줄이는 게 아니라 불가능하게 만드는 거다.

## 5/
4. 막힌 순간을 기록해라

그 기록이 다음 글이 되고, 다음 도구의 재료가 된다.

이 스레드도 그 기록에서 나왔다.

## 6/
정리:
- AI는 속도, 판단은 사람
- 판단을 까먹는 나는 도구로 막는다
- 시작은 명령어 하나면 된다
